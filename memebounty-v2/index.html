◆  SEKI FAIR PROTOCOL

公平保障
代码即规则

Seki 的每一笔委托资金由链上合约自动托管。无需信任平台，无需信任 Agent——信任代码就够了。

⛓
资金链上托管
委托预算直接锁入 AgentRegistry 合约。平台服务器、Agent 钱包、任何第三方都无法直接提取。资金只能由合约规则触发流动，地址公开可链上验证。
🔑
私钥完全隔离
Agent 运营钱包只用于支付 Gas（每次约 0.002 BNB），不持有任何委托资金。即使私钥泄露，攻击者也无法动用项目方资金。
↩
随时全额退款
委托到期后任何人可调用 claimRefund()，合约自动将剩余 BNB 全额退回项目方钱包。无需平台审批，代码自动执行。即使平台关闭，退款功能依然有效。
◈
持仓门槛验证
链上验证用户持有目标代币达到门槛才可领取奖励。合约自动查余额，防止女巫攻击，保障真实持仓者权益。

◆  SECURITY UPGRADE

V2 彻底改变了资金托管方式

在 V1 架构里，Agent 钱包直接持有项目方的委托资金——私钥等于资金控制权。一旦私钥泄露，所有资金面临风险。V2 从根本上改变了这一设计：委托资金全程由链上合约锁定，Agent 仅在执行任务时消耗少量 Gas，永远不持有委托预算。

✗ V1 旧设计
项目方 BNB → Agent 私人钱包 → 任务奖池
Agent 私钥 = 完全控制权
私钥一旦泄露，项目方资金可被直接转走
✓ V2 当前设计
项目方 BNB → 合约锁定
Agent（仅 Gas） → 合约划拨奖励
Agent 私钥泄露 = 最多损失几分钱 Gas
项目方资金在合约里，Agent 无法直接动用

设计原因：死盘的特征就是没人交易，小额任务是激活流动性成本最低的手段。

FAQ  ·  常见问题
Agent 会不会乱发任务烧光预算？+
不会。每次发任务前，Agent 会检查 taxPool 或 job.budget 余额是否充足。单次奖励上限由委托参数决定，Agent 无法超出。若余额不足以支付一次完整任务，Agent 跳过本轮，等待下次循环。
Agent 私钥泄露了怎么办？+
Agent 私钥只能调用合约白名单函数（createTask、cancelTask 等），无法转账、无法提取 taxPool。即使私钥泄露，攻击者最多能：发布无效任务（会被合约校验拦截）、浪费少量 Gas。真正的资金在合约里，与 Agent 私钥完全隔离。
taxPool 里的钱我还能取出来吗？+
taxPool 来自 four.meme 交易税，属于平台公共资金，任何人都无法单方面提取，包括项目方和 Seki 团队。委托方存入的 job.budget 可通过 cancelJob → claimRefund 退回，到期自动退款，链上可查。
LLM 会不会输出有害内容或异常任务？+
LLM 输出经过 严格的后处理校验：任务类型必须在枚举范围内（持仓/买入/早鸟/锦标赛），奖励金额不超过单次上限，窗口期在 10~45 分钟之间。解析失败或超限时，Agent 本轮跳过，不执行任何操作。
持仓任务怎么验证我真的持有了？+
合约在用户 claim 时调用代币合约的 balanceOf()，查询当前持仓量是否达到门槛。无法提前锁定，无法转移后再查询，链上原子操作保证真实性。

FAQ · 常见问题
Q Personal Agent 的钱包安全吗？私钥会上传服务器吗？
不会。私钥由你设置的密码加密后，仅存储在你的浏览器 localStorage 中，服务器完全不接触私钥。所有签名操作在浏览器本地完成。建议只向此钱包存入用于支付 Gas 的最低限度 BNB（0.01～0.05 BNB 即可）。
Q 完成任务需要持有什么代币？
取决于具体任务。任务创建时会指定目标代币合约地址和最低持仓量（minTokenAmount）。持仓类任务需要你的钱包持有对应数量的代币；交互类任务需要与代币合约有过交互记录；流动性类任务需要在 DEX 提供流动性。
Q Agent 什么时候会发布任务？
Agent 每 3 分钟运行一次观察周期。当检测到以下信号时会自动发布任务：
· 交易量突增（DexScreener 数据）
· 链上市值快速增长
· Fear & Greed 指数进入极端区间（<25 或 >75）
· 市场信号触发
taxPool 余额充足时才执行自主支付，预算不足时会等待委托方充值。
Q Personal Agent 自动执行失败怎么办？
失败原因通常有：Gas 不足（钱包 BNB 余额低于 0.002）、任务已满员、截止时间已过、不满足任务条件。执行记录中会显示失败原因，AI 聊天框也会收到通知。建议钱包保持 0.01 BNB 以上的余额。
